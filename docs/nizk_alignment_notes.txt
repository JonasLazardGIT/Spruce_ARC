NIZK ALIGNMENT NOTES

This file collects implementation-alignment notes that do not belong inside the
paper text.

1. Carry notation

The paper now uses C^J, J_0, and J_1 for the centering carry surface in the
pre-sign statement. The codebase still names the same objects C^K, K0, and K1
on the issuance path.

Relevant code paths:
- PIOP/credential_rows.go
- PIOP/credential_constraints.go
- issuance/flow.go

2. Pre-sign witness surface versus older docs

The pre-sign proof commits more than carrier rows. The live pre-sign witness
surface contains:
- 5 carrier rows
- 9 decoded alias rows
- 4 transform-domain replay aliases

This is the surface exercised by the pre-sign builder and verifier tests. Older
descriptions that say issuance commits only carriers are obsolete.

Relevant checks:
- go test ./PIOP -run TestCredentialPreSignProofVerifies -count=1

3. Showing replay surface

The showing proof commits:
- signature limb rows
- the message and centering carriers
- replay aliases hat(T_0), hat(m), hat(R_0), hat(R_1)
- packed PRF companion rows

The showing proof does not commit packed signature source rows, intermediate
signature hats, semantic U rows, or a semantic T row.

Relevant checks:
- go test ./PIOP -run TestTransformBridgeReplaySurfaceUsesOnlyTHat0 -count=1
- go test ./cmd/showing -run TestShowingV3TranscriptRegression -count=1
