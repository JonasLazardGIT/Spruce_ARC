\section{Implementation Results}
\label{sec:implementation-results}

We implemented ARC-SPRUCE in Go. Ring arithmetic and NTTs are provided by Lattigo~v4.1.1, while the
credential and commitment logic, the retained SmallWood proof stack (PIOP together with DECS/LVCS), and the
Poseidon2-like PRF are implemented in custom Go code over the same field \(\Fq\). Table~\ref{tab:implementation-results}
reports the single measured issuance instance and the retained reduced-replay showing instance used by the current
implementation. Benchmarks were obtained on a Linux \texttt{x86\_64} host with Go~1.26.0 on an Intel i7-10875H.

For the measured parameter sets, issuance is smaller because it only certifies the public-target pre-sign relation.
Showing additionally carries the reduced replay surface, the PRF companion constraints, and a nested hidden shortness
proof, so its size is dominated by the shortness bucket together with row-opening and quotient-related \(Q\) material.
Beyond the shortness proof, the remaining replay burden is concentrated in the PRF checkpoint and final-tag rows and in
the two source-product rows induced by the cleared \texttt{bb\_tran} relation. Conservatively, the retained reduced
showing instance stays below \(26\)~KB in paper-transcript size and verifies in under \(0.1\)~s, while issuance proving
stays below \(0.2\)~s for the measured instance.
